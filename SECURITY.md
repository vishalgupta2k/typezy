# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in typezy, please report it by:

1. **DO NOT** open a public GitHub issue
2. Email the maintainer or open a private security advisory on GitHub
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Response Timeline

- **Initial response**: Within 48 hours
- **Status update**: Within 7 days
- **Fix release**: Depends on severity (critical: ASAP, high: 7 days, medium: 30 days)

## Security Best Practices

This package:
- ✅ Has zero runtime dependencies (minimizes supply chain risk)
- ✅ Contains only pure functions (no side effects)
- ✅ Does not access filesystem, network, or environment variables
- ✅ Is fully open source and auditable

Thank you for helping keep typezy secure!
